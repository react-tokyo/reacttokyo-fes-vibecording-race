補足コメント

★ Insight ─────────────────────────────────────

良い点:

- sharedパッケージによるフロント/バック間の型共有は、このプロジェクト規模における最も賢い設計判断の一つ。API契約の一貫性を型レベルで保証している
- DrizzleスキーマのuniqueIndex("unique_follow")や外部キー制約など、データ整合性をDB層で担保している点が堅実
- TanStack QueryのqueryKey設計とinvalidateQueries戦略が体系的で、データ同期の仕組みが明確

主要な問題点:

- backend/src/index.ts:16 の app.route("/api", postsRoute) により、POST /api でも投稿が作成できてしまう意図しないエンドポイントが存在
- フロントエンド280文字 vs バックエンド140文字のバリデーション不整合は、ユーザー体験に直接影響するバグ
- テスト不在は品質保証の観点で大きな減点要因
